    Index

    Basic information for commands

    Only brief usage of each commands will be covered. See the man page or tldr for detailed usage.

    1. rg

    递归匹配字符串（更快，更简洁）

    2. grep

    匹配字符串

    3. tldr

    （too long, don’t read) 用户友好的使用手册

    4. fuck

    纠正并执行上一条错误命令

    5. find

    按一定条件查找文件（并可以指定执行一定的操作）

    6. fd

    查找文件，支持彩色

    7. history

    返回执行过的命令记录( | grep string 过滤)

    8. Ctrl + R

    输入字符串，匹配已执行的命令

    9. fzf

    将stdout作为输入，在其内容中支持动态匹配查找

    10. tree /broot

    按树状图打印文件，broot功能更多，支持缩略，gitignore，预览，匹配查找

    11. Shellcheck

    检查语法错误

    12. journalctl

    打印系统日志

    13.

    对于远程主机，可以将grep筛选过程均在远程完成，（用“”将命令打包，ssh rmtpc “cmd”）

    14. less

    一种文件（简略信息？）查看方式

    15. sed

    对输入流进行一定的处理(删除、替换）再输出

    16. sort

    对输入流进行排序

    17. uniq

    uniquing 排序过的输入（-c计算每个重复的次数）

    18. tail & head

    仅仅输出输入中的开头/结尾n行

    19. awk

    基于列（空格隔开的字符串叫做一列）的操作 （匹配加操作）

    20. paste

    将若干行输出合并起来输出（paste -sd，加’,'进行输出）

    21. bc

    (-l) 计算字符串形式的算术表达式

    22. nohup

    启动不依赖于终端的命令（关闭当前终端仍然可以运行）

    hup是指传输给进程的hangup信号，在关闭终端时发送。

    对于非nohup进程，hangup == kill；对于nohup进程，该信号被忽略。

    23. jobs

    查看当前终端的所有进程

    24. bg / fg

    ctrl + z是放入后台挂起，bg 工作号 可以让它后台执行， fg 工作号是拉回前台

    25. kill

    不仅可以终止进程，还可以向进程发送暂停等信号（—KILL指传统意义的kill）

    26. tmux

    在shell中创建tmux终端，可以创建多个，在其中也可以运行命令

    session(一个tmux终端) -> windows(一个窗口（tab）) -> panes（一个分区）

    27. pgrep/pkill

    匹配字符串查找，终止进程

    28. wait

    wait id (挂起当前会话）然后再执行后续

    可以通过；连接，实现进程的先后执行

    29. ssh

    ssh foo@bar.mit.edu

    30. du / ncdu

    显示硬盘占用情况

    31. lsof（list of open files)

    可以通过文件判断进程的情况,或者查找到进程

    lsof -n (no host name) 可以加快速度（浏览器打开情况下），用strace检查过程，发现sendto函数？卡住。

    原理可能是浏览器的打开文件一直在改变。
