keycode

Posted on 2022-08-06 Symbols count in article: 471 Reading time ≈ 1 mins.

Record some odd default setting of the keyboard for further configuration.

keycode -> sysfunc
两种修改方式：/usr/share/X11/xkb/keycodes/evdev + sudo dpkg-reconfigure xkb-data
和修改~/.Xmodmap（通过xmodmap和xev）亲测有效为前者（重启）
1.win的keycode为133,caps为66,esc为9.caps与esc互换。（win被映射到meta/super）
2.对于fx，现问题是他们没有keycode，按键结果为其他键的组合键或者无。
3.终端的设定：SIG只识别ctrl，如果ctrl c被占用，会变为ctrl+shift实现信号发送。
4.终端是独立于外部的键盘设定。
5.实现目标：win+实现复制粘贴等ctrl操作，同时在终端中保持ctrlc、z信号。
6.先决否定：如果互换win，ctrl，那么终端中就一定要按win + c，z（SIG无法修改），且此时，终端中一定不能保持和外部一样的复制粘贴
